Bạn là một Frontend Architect.

Hãy chuyển đổi Business Spec dưới đây thành UI Spec chi tiết.

Yêu cầu:

1. Xác định Page / Screen

* Route (đường dẫn)
* Mục đích của màn hình

2. Xác định UI Components

* Liệt kê các component cần có (input, button, banner, text...)
* Tổ chức theo cấu trúc component cha – con

3. Xác định Layout

* Bố cục (center, full screen, grid…)
* Vị trí các thành phần trên màn hình

4. Xác định UI Behavior

* Khi user tương tác (click, nhập liệu…) thì xảy ra gì
* Hiển thị lỗi / loading / success

5. Xác định Style (dựa trên Business Goal)

* Màu sắc
* Typography
* Cảm giác thiết kế (modern, tối giản, công nghệ…)

6. Xử lý Edge Cases

* Nếu lỗi xảy ra (mạng, dữ liệu, ảnh…) thì UI hiển thị thế nào

7. Mapping rõ từ Business Spec → UI Spec

* Mỗi yêu cầu trong Business Spec phải có phần thể hiện trên UI
